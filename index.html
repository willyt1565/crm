Team CRM · Supabase Setup

One-time setup by an admin. Once connected, any team member can create their own account at this URL.

1
Create a free Supabase project
Go to supabase.com → New Project → name it npm-crm → choose a region → Create new project. Wait ~60 seconds.
2
Disable email confirmation (important)
Go to Authentication → Settings → turn off Enable email confirmations → Save. This lets team members sign up instantly without clicking a confirmation link.
3
Run this SQL to create your tables
Go to SQL Editor → New query, paste and run:
-- Profiles (auto-created on signup) create table profiles ( id uuid references auth.users primary key, full_name text, email text, created_at timestamptz default now() ); alter table profiles enable row level security; create policy "team can view profiles" on profiles for select using (auth.role() = 'authenticated'); create policy "users update own profile" on profiles for update using (auth.uid() = id); -- Auto-create profile on new user signup create or replace function handle_new_user() returns trigger as $$ begin insert into profiles (id, full_name, email) values ( new.id, new.raw_user_meta_data->>'full_name', new.email ); return new; end; $$ language plpgsql security definer; create trigger on_auth_user_created after insert on auth.users for each row execute procedure handle_new_user(); -- Leads create table leads ( id text primary key, data jsonb, assigned_to uuid references profiles(id), created_by uuid references profiles(id) default auth.uid(), updated_at bigint ); alter table leads enable row level security; create policy "team leads" on leads for all using (auth.role() = 'authenticated') with check (auth.role() = 'authenticated'); -- Tasks create table tasks ( id text primary key, data jsonb, created_by uuid references profiles(id) default auth.uid(), updated_at bigint ); alter table tasks enable row level security; create policy "team tasks" on tasks for all using (auth.role() = 'authenticated') with check (auth.role() = 'authenticated');
4
Enter your project credentials
Go to Project Settings → API → copy your Project URL and anon public key.

Share this URL with your team. Anyone who opens it can create their own account.
